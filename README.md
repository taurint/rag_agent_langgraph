## 🎯 Что делает агент

Риелтор или клиент задаёт вопрос по недвижимости — агент ищет по базе знаний и отвечает. Если информация не найдена — честно эскалирует оператору.

## 🧠 Архитектура
ЗАПРОС ПОЛЬЗОВАТЕЛЯ
"Как расторгнуть договор аренды?"

RETRIEVAL — поиск релевантных чанков

Dense поиск (FAISS)
Находит чанки по смысловому сходству
Эмбеддинги: OpenAI text-embedding-3-small (1536D)

Sparse поиск (BM25)
Находит чанки по ключевым словам

Hybrid Fusion
Смешивает результаты dense и sparse
Веса: 0.7 (dense) и 0.3 (sparse)

AGENT — LangGraph граф

retrieve
Ищет релевантные чанки из базы знаний
Считает confidence (средний score топ-3)

validate
Проверяет confidence > 0.5
Если да → answer
Если нет → escalate

answer
Генерирует ответ через GPT-4o-mini
Отвечает строго на основе контекста

escalate
Переводит запрос на оператора
Сообщает клиенту о передаче вопроса

MONITORING — LangFuse

Трейсинг каждого запроса
Latency на каждом этапе
Результаты golden set


## 📦 Стек

- **LangGraph** — граф агента с ветвлением
- **FAISS** — векторная БД для dense поиска
- **BM25** — sparse поиск по ключевым словам
- **OpenAI API** — GPT-4o-mini (генерация) + text-embedding-3-small (эмбеддинги)
- **LangFuse** — трейсинг и мониторинг
- **Google Colab** — среда разработки и демонстрации

## 📊 Метрики (golden set из 7 вопросов)

| Метрика | Значение |
|---|---|
| **Точность (keyword match)** | 71% |
| **Точность (фактическая)** | 100% |
| **Эскалаций (не в тему)** | 1 из 3 тестовых |
| **Средняя latency (ответ)** | ~1.3 сек |
| **Средняя latency (эскалация)** | ~180 мс |
